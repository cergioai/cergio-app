Recommend flow: unified (contacts + manual) + Profile v3 + Earnings tabs

Three user-driven cleanups in one push. (Profile v3 + Earnings tabs were
queued in the previous Mac-locked window; the Recommend fix is the new
one and rounds out the signed-in UX pass.)

1. RECOMMEND FLOW — unified, no more dual-path popup

Tarik's audit: "recommend flow is wrong.. when you pick someone to
recommend, that's the services you're recommending not the person
you're recommending to.. the write a recommendation link is redundant..
remove... can either recommend from contacts or recommend a person by
typing their info (but can merge both)."

Old: RecommendServicePopupScreen showed two buttons — "Recommend from
contacts" (route → multi-select contacts picker that looked like Invite)
and "Write a recommendation" (route → free-form form with contacts-only
autosuggest, no manual entry). The split confused the model.

New: ONE screen at /invite/recommend with three merged inputs:

  ┌── Who are you recommending to? ────────┐
  │  [ From contacts × ]   ← only shown    │
  │                           when picked  │
  │  [ Friend's name        ]              │
  │  ↪ autosuggest dropdown (tap → fills)  │
  │  [ Phone        ] [ Email           ]  │
  │  Either one is enough.                 │
  └────────────────────────────────────────┘

Pick a contact → name + phone + email all autofill, subtle "From
contacts" pill appears. Or type all three manually for someone who
isn't in your contacts. Either path → same blurb step → same submit
(notify-user + recommendations row).

3. EARNINGS — Referrals vs Client bookings tabs

Tarik: "we forgot to build a track earnings from services for service
providers". Service providers earn from TWO streams — referral
commissions and revenue from their own services. Were merged into one
list, invisible as separate business lines.

The earnings.kind enum already supported the split; we just weren't
surfacing it. Now a 2-tab segmented control gates the ledger:

  Referrals  $42      ←  kind in (invite, spotlight)
  Client bookings  $1,250  ←  kind = booking

Providers default to Client bookings; consumers + Connectors default
to Referrals. First explicit tap is sticky. Per-tab empty state with
appropriate CTA (List a service / Invite a friend).
