fix(vercel): SPA rewrite so /auth /home /u/:id etc. resolve on cergio.ai

Direct routes (e.g. https://cergio.ai/auth) were returning Vercel's
default 404 because the build output is a single-page React app and
there was no rewrite rule. Every non-index URL needs to be sent back
to /index.html so React Router can take over.

Add vercel.json at the project root with the standard SPA fallback:

  {
    "rewrites": [
      { "source": "/(.*)", "destination": "/index.html" }
    ]
  }

This is the recommended Vercel config for any Vite + react-router app
and is what's referenced in GO_LIVE_CHECKLIST.md §1 implicitly.

Unblocks: Continue-with-Google round trip (OAuth returns to
/home which previously 404'd), Forgot-password reset link (/auth?reset),
Friend invite reco link (/u/:id), and every other deep link.
