ROOT CAUSE (proven via in-page A/B control banner):

  count (React-flow):           0   ← stale closure of listServices
  count (fresh import control): 1   ← Maria's Sparkling Home Cleaning
  fresh first: Maria's Sparkling Home Cleaning

ResultsScreen's `import { listServices } from '../lib/api'` binds at
mount time. Vite HMR updates api.js but does NOT re-bind imports in
already-mounted components. After any listServices edit, the running
ResultsScreen keeps calling the OLD function. Caused a 2-day debug
loop with fixes that worked in JS probes but not in the UI.

FIX

Replace the static import with a dynamic import inside the search
useEffect:

  const { listServices } = await import('../lib/api');
  const { data, error } = await listServices({ ... });

Every search re-resolves the module → latest api.js code → no stale
closures. Production builds unaffected (modules stable, no HMR).

Static import kept as a CERGIO-GUARD comment so a future refactor
doesn't reintroduce it.
