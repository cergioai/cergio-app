design: PDP pixel-comp pass + logo full-round flower + wordmark inline

Big design-comp pass against Tarik's PDP/SRP mockups. He asked for a
"rigorous process to put together a design spec that matches these,
then execute" — so this lands the tokens + design-spec deltas FIRST
and then applies them across the entry surfaces and the PDP.

Design tokens
- `line` (#EFE7D6) — ultra-thin cream-tinted hairline. Added to
  tailwind.config.js + design-spec.md. Use it (border-line) for any
  "barely there" separator on cream pages. `bdr` (#E5E5E3) stays for
  borders on white surfaces.
- design-spec.md now documents:
    • PDP hero pattern (360px, ruler at bottom, 5 segments, dark-
      translucent icon circles for back / heart / Connector / share /
      mute).
    • Offering card pattern (white bg + border-line, vivid-green
      "$X/session • Ymins" price format, vivid-green `% off` pill).

PDP (ServiceDetailScreen.jsx)
- Hero 280 → 360 to match the mockup aspect.
- Story-progress ruler moved TOP → BOTTOM (5 segments, leading
  segment full white).
- Top-left back arrow (subtle dark circle).
- Top-right control trio added: heart, Connector badge, share — all
  in matching dark-translucent circles, share fires the Web Share
  API when available.
- Volume icon moved BOTTOM-LEFT (matches the mockup). Caption line
  sits just above the ruler.
- Section divider above "Book {Name}" switched from `border-bdr` to
  `border-line` so it reads as a hairline on cream, not a contrasty
  rule.
- Offering cards refined:
    • bg-white + 1px border-line (selected = 1.5px green ring, no
      fill swap — kept surface weight constant).
    • Price format "$X/session • 120 mins" (vivid green price, b3
      duration).
    • New `discount_percent` field support — solid green pill at
      bottom-left when set ("20% off"), per the Cut and Color
      mockup tile.

Full-round flower (LeafLogo.jsx)
- Tarik (#B): "make the still non animated state open into a full
  round flower... not perfectly symmetrical but full round".
