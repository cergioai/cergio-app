Search: strict provider_type match end-to-end + locked with two new invariants

Real bug, fixed via observability-first (no more patches):

1. Parser path: the local PROVIDER_TYPE_MAP (60 phrases, source-of-truth
   for offline-fallback resolution) now overrides the Claude resolver when
   it has a concrete answer. Claude was returning synonymic-but-wrong
   strings ("Cleaning Service" instead of "House Cleaner"), which broke
   strict matching on the seed's `taxonomy_provider_type` column.

2. listServices proximity branch: `services_near` RPC returns ONLY the
   proximity-relevant columns (id, title, location_text, distance) —
   NOT `taxonomy_provider_type` or `taxonomy_offering_id`. The previous
   client-side strict filter compared `s.taxonomy_provider_type ===
   'Plumber'` on the raw RPC result, but `taxonomy_provider_type` was
   undefined for every row → every filter excluded every row.

   Fix: after services_near returns the in-radius IDs, hydrate full rows
   from `services` table including taxonomy_* columns + status='listed',
   then run the strict provider_type filter. One extra round-trip; the
   spec's "only the right provider type surfaces" guarantee actually
   holds now.

   Confirmed live via Chrome JS probe — same code path the UI uses:
     unclog my toilet     → Plumber          → J. Chen Plumbing      (6.8mi)
     deep cleaning        → House Cleaner    → Maria's Sparkling
     personal chef        → Personal Chef    → Chef Marcus
     hairstylist          → Hairstylist      → Jordan
     personal trainer     → Personal Trainer → Tasha Fit
     babysitter           → Babysitter       → Ana

3. Display copy: "near me" no longer pluralizes to "near mes" (added
   "near\s" to the stopAt regex in serviceNoun.js).

4. New invariants in qa.mjs:
   - #13 canonical-query-resolves: 23 canonical phrases must resolve to
     the expected provider_type via the local map. Catches taxonomy drift.
   - #14 proximity-hydrates-taxonomy: any code that filters by
     `taxonomy_provider_type` MUST be preceded by a .from('services')
     .select() that includes that column. Catches the exact bug above
     from ever returning.

5. Vite hardened: strictPort=true + Start Cergio.command kills stale
   process on 5173 before launch, so the dev server can't drift to
   5202 and trip the Google Maps referrer check.
