fix(tiktok-oauth): sign in returning TikTok users instead of failing

TikTok login failed with "User create failed: A user with this email address
has already been registered" for returning users. Root cause: the find-or-create
logic used admin.listUsers() which is PAGINATED (first ~50 users). Once the
project grew past one page, a returning TikTok account on a later page was
missed, so the code tried to createUser again → Supabase rejected the duplicate.

Fix: stop relying on a paginated first-page lookup. Attempt createUser, tolerate
the "already exists" error, and read the user id back from generateLink (which
resolves the user for the synthetic email whether just-created or pre-existing).
Profiles upsert guarded on a resolved userId. Login now works for both new and
returning TikTok accounts regardless of total user count.
