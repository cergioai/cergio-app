Add profiles.follower_count + render on GOAT activity cards

Tarik approved the follow-up: real follower counts on the Activity
feed GOAT-share cards, instead of the count-free fallback we shipped.

DB
  ALTER TABLE profiles
    ADD COLUMN IF NOT EXISTS follower_count integer NOT NULL DEFAULT 0;

Migration is in "Apply Follower Count Migration.command" (Mac-side
script, follows the same pattern as Apply Requests Migration.command:
runs the ALTER via Supabase Management API, verifies column exists,
prints current Connector counts so Tarik can see who's set vs default).
Idempotent — safe to re-run.

API
  src/lib/api.js → listGoatShares() now selects follower_count on
  the recommender profile and surfaces it on the returned row at
  row.recommender.follower_count.

UI
  src/screens/ActivityScreen.jsx → GoatShareCard headline switches
  based on the count:
    followers > 0  →  "{Owner} was shared to {N,locale} followers"
    followers === 0 →  "{Owner} was shared on Cergio" (unchanged)
  Numbers are toLocaleString'd so they render as "45,414" not 45414.
  Never fake — if the column is 0 we don't show "0 followers".

How to populate (Tarik, in Supabase SQL editor):
  UPDATE profiles SET follower_count = 45414 WHERE display_name = 'Sabir';
  UPDATE profiles SET follower_count = 135572 WHERE display_name = 'Jackie';
  -- etc.
