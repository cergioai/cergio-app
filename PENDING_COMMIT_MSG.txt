Recommend service-type field + Consumer PDP + Earnings degree-of-separation

Three audit slices in one push. The Recommend gap (no service-type
field), the missing Consumer PDP screen (the "Jennifer Leighton" view
in the Figma reference), and the missing degree-of-separation on the
earnings ledger.

1. RECOMMEND — service-type field

Tarik: "this is missing the service type (plumber etc) field."

The unified Recommend form had Name / Phone / Email / Blurb but no
field for WHAT was being recommended — sent the recipient a generic
"a service on Cergio" blurb with no category.

Fix: new "What service are you recommending?" input above the recipient
section, with a 16-entry datalist (Plumber, Tutor, Cleaner, Photographer,
Hairstylist, Pet Sitter, Mover, etc). Required to submit.

  • Passes through to notify-user data.service_title so the email/SMS
    reads naturally: "Maya recommends Plumber: …" instead of
    "Maya recommends a service on Cergio: …"
  • Prepended to the saved recommendations.message: "[Plumber] blurb…"
  • Submit button label adapts: "Pick a service type first" before the
    type is filled in

  src/screens/RecommendServiceFormScreen.jsx
    • New serviceType state + validation
    • Inline datalist for autocomplete suggestions

2. CONSUMER PDP — ServiceDetailScreen (audit slice 2)

Audit (2026-05-29) found there was no consumer service-detail screen.
ResultsScreen jumped straight to /booking — skipping the trust step
entirely. New screen at /service/:serviceId.

Layout (matches the Figma reference):
  • Header: back + share
  • Cover image (or gradient fallback) — same cover_url path as cards
  • Title row: provider name + category + price
  • Recommended-by panel: avatar stack ("Recommended by Alex, Sam and 1 more")
    + 3 italic blurb rows with the recommender's display name
  • About this service
  • Fixed-bottom Book CTA — reuses the existing handleBook flow

Fast-path: ResultsScreen passes the full provider object +
recommenders via location.state so the PDP renders instantly without
a re-fetch. Cold-deep-link fallback queries services + recommendations
+ profiles by id.

3. EARNINGS — degree-of-separation pill per row (audit slice 3)

Audit found the earnings ledger merged direct $250-track payouts and
friend-of-friend $12.50 chain bonuses into identical-looking rows.
Now each referral row carries a small tier pill:

  Direct       — green pill (bg-gl / text-gd) — full $250-track payout
  Chain +5%   — amber pill (bg-warnBg / text-warnText) — FoF bonus

Classifier (earningTier(e)) prefers meta.tier if the stripe-webhook
writes it; falls back to an amount heuristic (≤$50 → chain). Spotlights
+ bookings render no tier pill (not applicable to those kinds).

  src/screens/EarningsScreen.jsx
    • earningTier helper
    • Per-row pill rendered next to the title
