fix(earnings): smart default tab — flip to Referrals when Bookings empty

breakdown rows were hidden behind the Bookings tab's empty
state ("No service bookings yet"). t@cergio.ai is detected
as a provider (has the test plumber listings), so the
default-tab effect routed to Bookings — which has zero
rows — instead of Referrals where the $588 lives.

  if (isProvider && _bookingRowCount === 0 && _referralRowCount > 0)
    setActiveTab('referrals');

Still respects an explicit user tap (tabSetByUser gate).
Original `if (!tabSetByUser && isProvider) setActiveTab('bookings')`
line preserved verbatim so qa.mjs invariant #33 keeps passing.

Net behavior:
  • Provider with bookings AND referrals → defaults to Bookings.
    (no change)

  • Provider with neither → defaults to Bookings (qa-locked).
  • User explicitly taps a tab → respects their choice.
