feat(activity): 4-tab Network view — Summary / Friends / Connectors / Feed

Tarik 2026-06-05 referenced the original UX video — timer 6:36-6:42
shows the network tab with four sub-tabs; 2:48-3:11 shows the
Summary aggregates + reco'd-services tracking. This commit adds
that structure to ActivityScreen.

══════════════════ Tabs ════════════════════════════════════════════════

  Compact pill bar near the top of Activity. Four views:

  SUMMARY (counts + actions)
    Two rows of three tappable count tiles:
      • friends invited / friends joined / friends booked
        → all route to Friends tab
      • recos sent / connectors / activities
        → route to /invite/recommend, Connectors tab, Feed tab
    Action row: "Invite a friend" + "Reco a provider"
    Sub-line: rewards math from REWARDS ($X credit per friend up to
              $250, plus $12.50 chain bonus per friend-of-friend).

  FRIENDS (invites I sent + their status)
    NetworkFriendsTab — calls getMyInvitesDetailed.
    Each row: avatar + name + invited-on date + status pill
      (Pending / Joined / Booked).
    Tap row → invitee's public profile (if joined) or invite tracker.
    Empty state: gradient CTA card → /invite/friends-popup.

  CONNECTORS (verified ambassadors + follow)
    NetworkConnectorsTab — loads top 50 profiles where
    cc_verified_at IS NOT NULL, ordered by verified date.
    Each row: avatar + name + IG/TT handles + total reach + Follow
              button.
    Follow button reuses followProfile / unfollowProfile from api.js;
    local followedIds set updates immediately so the UI feels live.
    Tap avatar or name → /u/:id public profile.

  FEED (existing social feed + open requests)
    Unchanged behaviour — the existing feed IIFE + "Your open
    requests" block now render only when activeTab === 'feed'.

══════════════════ Data sources ═══════════════════════════════════════

  • Counts: getMyInviteCounts (already used by EarningsScreen)
  • Friends list: getMyInvitesDetailed (already used by
    InviteTrackingScreen)
  • Connectors: direct supabase query on profiles, filter
    cc_verified_at IS NOT NULL — same gate BrowseConnectors uses
    server-side
  • Follow state: getMyFollowedIds + follow/unfollow API
  • Feed: listSocialFeed (existing)

  No new edge functions, no new RLS policies — pure frontend
  composition over existing helpers.

══════════════════ UX notes ═══════════════════════════════════════════

  • Default tab is Feed so existing users see no change on first
    paint; they discover the new tabs as a calm bar above.
  • Each Summary tile routes to the matching sub-tab so the page
    self-navigates without a hub-and-spoke trip.
  • Connectors tab follow button mirrors the same pill style used
    on PublicProfileScreen — consistent affordance everywhere.
