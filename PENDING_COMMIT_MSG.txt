fix(screenshot-qa): resolve playwright via createRequire, not NODE_PATH

First run of Screenshot QA.command hit ERR_MODULE_NOT_FOUND on the
`import { chromium } from 'playwright'` line. Root cause: Node ESM
resolution does not honor NODE_PATH (which CJS does honor) — the
.mjs script couldn't see the playwright install at
~/.cergio-playwright/node_modules even though the launcher set
NODE_PATH to that directory.

Fix: drop NODE_PATH; export CERGIO_PW_DIR pointing at the absolute
playwright dir; in the script, use createRequire(import.meta.url)
to require playwright as a CJS module from that absolute path.
Playwright's CJS entry exports `{ chromium, firefox, webkit }`, so
the destructured shape works under both module systems.

Verified end-to-end: relaunched Screenshot QA.command after the
edit; all 10 canonical screens captured, gallery opened in
default browser at 5/31/2026 11:47:31 PM.

No code path inside cergio-app changed — this is a tooling fix
for the design-qa harness only. qa.mjs untouched, still 41/41.
