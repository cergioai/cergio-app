Free-service barter loop (SPEC-47): schedule confirm → IG post → provider accept → gate

Implements Tarik's User Flow / SVP Flow board end-to-end:

1. ScheduleSheet (new) — calendar + time slots + Done. Every real
   booking (PDP + on-demand, free + paid) now opens it so the user
   confirms day/time; bookings carry schedule_confirmed_at instead of
   the silent "+24h placeholder".

2. No auto-confirm on submission — free bookings AND demo-mode paid
   bookings stay PENDING until the provider accepts (SVP "Accepting
   Free Service request"). BookingScreen says "Request sent!" with
   honest copy; consumer sees "Awaiting confirm" in Inbox → Upcoming.
   Provider accept now notifies the consumer (notify-request
   event=booking action=accepted; email reminds free Connectors to
   post after the job).

3. Post → review loop: Connector "Mark IG post done"
   (MarkBookingPostedModal → markBookingPosted: post_url + posted_at,
   clears prior flags) → provider notified → Inbox → Upcoming shows
   "View their post / Accept post / Something's wrong" (inline flag
   reason, no window.prompt). Accept → post_confirmed_at + status
   completed → barter complete; flag → Connector notified with reason
   and can resubmit.

4. THE GATE — getOutstandingFreeBarter(): a Connector with an
   accepted free booking whose post isn't provider-confirmed cannot
   order another free service. Checked in handleBook before the
   schedule sheet; releases ONLY on provider confirmation.

5. Activity feed: posted barters surface as kind 'barter'
   ("X bartered {service} from Y for an IG spotlight ✓ View post →"),
   same party-scoped visibility as spotlight events.

6. Upcoming / Past tabs in Jobs are real now — both roles, barter
   CTAs, completed barters land in Past. Inbox dot also fires when a
   post lands for provider review.

post_confirmed / post_flagged) — needs redeploy.
