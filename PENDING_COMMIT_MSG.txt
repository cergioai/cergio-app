fix(tiktok-signin): full-page redirect flow (popup callback was served as text/plain)

Root cause of "TikTok login doesn't work": the popup callback returned an HTML
page whose inline script completed sign-in (postMessage to opener + window.close).
But the Supabase gateway served that page as text/plain, so the browser showed
raw source (garbled, mojibake) and NEVER executed the script — sign-in never
completed even though the OAuth + user lookup succeeded (ok:true in the payload).

Fix — switch TikTok sign-in to a standard top-level redirect (gateway-independent,
like Google sign-in):
- AuthScreen: click "Continue with TikTok" → window.location.href = TikTok auth
  URL (no popup, no postMessage dependency).
- tiktok-oauth callback (signin mode): instead of returning the HTML/JS page,
  302-redirect the browser straight to the Supabase magic link, which verifies
  and lands on <origin>/home logged in.

(Prior commit baae65a fixed the paginated listUsers create-vs-signin bug; this
commit fixes the page-never-rendered/script-never-ran bug that was the actual
blocker.) Note: TikTok "link/connect" mode still uses the popup page — separate,
lower-priority; can get the same treatment later.
