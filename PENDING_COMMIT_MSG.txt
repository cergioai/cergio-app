fix(earnings): unblock $0 ledger + (i) embedded inside toggle button

═══════════════ (1) Earnings shows $0 for t@cergio.ai ═══════════════

Tarik 2026-06-03 — Earnings page rendered $0 even though the
referral audit showed t@cergio has 6 invite payouts worth $838.

Root cause in src/lib/api.js → getMyEarnings():

  .select('id, kind, source_id, amount_cents, currency, status,
           created_at, meta')
  .order('created_at', { ascending: false })

But the actual `earnings` table column is `occurred_at`
(confirmed via Test Referral Payments.command schema probe).
PostgREST silently returns a 400 on the unknown column → data
is null → setEarnings(data || []) lands as an empty array →
totals.all = 0 → "$0".

Fix:
  • Select + order by `occurred_at`.
  • Map `occurred_at` → `created_at` in the returned shape so
    EarningsScreen and other consumers don't need to change.

Net behavior after refresh: t@cergio.ai will see all 6 invite
rows (4 direct $250s + 2 chain $12.50s) in the Referrals tab,
cleared/pending split matching section 3 of the audit.

═══════════════ (2) (i) icon was wrapping below toggle at narrow widths ═══════════════

Tarik: "the i is back at the bottom left.. place just to the left
of the arrow on top right."

v2's sibling-button layout still wrapped at the narrow phone-shell
width. Restructured so the entire toggle is ONE button containing

  • inline-flex items-center
  • whitespace-nowrap on the button
  • flex-shrink-0 on the (i) and chevron
  • the (i) is a clickable <span role="button"> nested inside the

    /rainmaker/apply instead of toggling the dropdown.

Guarantees the (i) stays inline with the chevron, no matter the
container width.
