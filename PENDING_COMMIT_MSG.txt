══════════════════ (a) Earnings cleanup ═══════════════════════════════

  1.  $838 inflation killed. ReferralsSummary now sums *capped*
      buckets ($250 per direct friend, $12.50 per chain path) instead
      of the raw ledger total. Seeded test data above the cap stops
      leaking into the headline.

  2.  Per-row "$250 → $0 left" math. Walks referralRows oldest-first,
      accumulates per bucket, stamps each row with capLeftCents +
      capReachedBefore. FriendTallyPill renders "$X left" (not the
      old "$X/$Y" lifetime-total). If the cap was already reached in
      a previous row, the pill reads "$0 left ✓" and the row caption
      says "cap was already maxed — $0 paid on this booking".

  3.  Top-3 leaderboard now uses the same per-friend capped sums so
      it reconciles with the hero total. No more "Sam: $838" when
      the cap is $250.

  4.  Top counts strip — 5 tappable tiles between the hero card and
      the tabs:
          Invited  Joined  Booked  Reco'd  Services
      Each tile is a real count (inviteCounts / recsCount /
      servicesCount). Tap = jump to the next-best action:
          Invited/Joined/Booked → /earnings/invites (filtered)
          Reco'd                → /invite/recommend
          Services              → /profile (if has service) else /list-service
      The strip itself nudges users toward more invites + recos
      without the user needing to know how to find those flows.

  5.  Reco link routing fix. "Or, recommend a service" on
      InviteFriendsScreen used to send the user to
      /invite/friends?mode=reco — a generic contact-list picker that
      didn't ask which service to recommend. Now routes directly to
      /invite/recommend (RecommendServiceFormScreen) which asks for
      service + contact in one place. Matches Tarik's spec verbatim.
