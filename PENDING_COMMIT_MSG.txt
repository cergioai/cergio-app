fix(ux): map picker exit controls + remove service-side promo on user Home

Tarik (two messages):
  1. "draw on may should have save or cancel so user can exit of that
     screen"
  2. "remove the 'service side from the user side (they'll know to
     switch to service side to list a service etc)"

ServiceAreaMapPicker — explicit Cancel + Save
  Before: footer had Clear (disabled when no shape) + Save (disabled
  when no shape with copy "Draw an area first"). The ONLY exit when
  nothing was drawn was the small ✕ in the header — easy to miss on
  mobile.
  After:
    [ Cancel ]   [ Clear (only when shape) ]   [ Save area ]
    • Cancel — always-enabled white pill, closes the sheet without
      saving. The labelled escape hatch Tarik asked for.
    • Clear — only renders once the user has drawn a shape (no longer
      a permanently-greyed-out button taking up space).
    • Save area — primary green pill, enabled the moment a shape
      exists. Copy switches between "Save area" and "Draw an area
      first" depending on hasShape.
  Header ✕ kept for users who reach for the corner; both exits land
  in the same `onClose` callback so behaviour is consistent.

HomeScreen — consumer-side "Become a Connector" promo removed
  Block that rendered when `intent === 'find'` showed a "Become a
  Connector — $X cash + free services" CTA above the footer. Tarik
  wants the consumer Home focused on FINDING services, not on being
  recruited to the service side. Block removed; a CERGIO-GUARD
  comment marks the spot so we don't accidentally reintroduce it.
  The Connector apply flow stays reachable from:
    • the (i) chip next to "Free for Connectors" in the search box
    • /about (Mission + Streams + (i) link)
    • /earnings/how (explainer + apply CTA)
    • the spotlight-intent equivalent promo (still rendered when
      intent === 'spotlight')
