Search: in-page diag banner when listServices returns empty

Quick visible diagnostic so users can screenshot the actual values
without opening DevTools. When ResultsScreen receives count: 0, a
yellow CERGIO-DIAG banner appears above the empty-state card showing:
  - provider_type
  - offering_id
  - category
  - lat/lng
  - originalQuery
  - count + first matches if any

One tap dismisses. Same data also still goes to console for ops.

Once the search bug is resolved permanently for real users, this
banner can be removed in one edit — it's gated on services.length===0
so it never fires on happy-path.
