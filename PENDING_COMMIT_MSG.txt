Provider request screen (SPEC-48): job details, approximate map, IG block, friends-in-common

Builds the "Accepting Free Service request" screen (Tarik flow board) so the
service provider sees everything needed to decide on an inbound free request.

RequestDetailScreen now renders, in mockup order:
  - Job details (title, free-for-Connectors pill, description, appointment)
  - Approximate-location card — "Map shows approximate location"; the exact
    address is shared only AFTER the user confirms the booking. No live map
    tile / no precise pin until confirmed.
  - Instagram block — requester handle + follower count + working
    "See Instagram" link (hidden when no connected handle).
  - Friends-in-common — mutual network connections with the requester,
    bucketed friends + Connectors (hidden when zero).
  - Free-marketing benefit subcopy + "Accept free request" CTA.

Data layer:
  - getBooking() now selects consumer.instagram_handle / instagram_followers
    / cc_verified_at for the IG block.
  - New getMutualConnections() computes mutuals over the canonical `network`
    graph (any edge, either direction), returning count + Connector count +
    a capped sample. No fabricated data.

No fake IG photo grid: the "+N more" thumbnail strip is gated on real
data.igMedia (populated once Meta Graph media access is approved) — never
placeholder tiles (SPEC-12).

Frozen as SPEC-48 in FROZEN_SPEC.md + enforced by qa.mjs #48.
