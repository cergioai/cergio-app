Fix: @vite-ignore so missing maps loader can't crash whole app

Same Vite crash Tarik just hit again. The previous "dynamic import"
fix wasn't truly dynamic — Vite's import-analysis still tried to
resolve the literal string `'@googlemaps/js-api-loader'` at dev-server
start and crashed because the package isn't installed yet on Tarik's
machine.

Adding /* @vite-ignore */ before the specifier tells Vite to skip
analysis entirely. The resolve now happens at RUNTIME, only when the
ServiceAreaMapPicker actually mounts. If the package is missing the
picker throws a friendly "run npm install" error — the rest of the
app keeps running.

Permanent fix path is still: Tarik double-clicks
"Reinstall Deps and Start.command" to install the package. This
patch just keeps the app from crashing in the meantime.
