/inbound: message uses the requester's raw request text

The message task was showing the parsed type ("Catering") because the field
preference favored `what` over the raw words. Now getInboundRequest also pulls
`query`, and the message task prefers `description || query || what` — so it
reads the requester's actual ask (e.g. "vegan Ecuadorian dinner for 6") rather
than the category. (This test request only has "Catering" as its raw text.)
