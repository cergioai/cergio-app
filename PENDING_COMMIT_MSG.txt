revert(results): remove category-nav rail + filter pill row — out-of-scope for design audit

Per Tarik (2026-06-02): "remove these tags from top of results...
All services / Free / Discounted / 🧺 Housekeeper / 🐾 Pet Sitter /
🔧 Handyman / 🏋️ Trainer / 🚗 Driver. they were in designs that
should not have been carried through (as I had asked for design
audit without changing that information architecture)."

The Phase 3d audit pass introduced two new navigation surfaces on
the Results screen — a horizontal category-nav rail and a filter
pill row — based on mockup-matching against Jennifer L's SRP. Tarik
clarifies that the audit was supposed to stay typography + brand
only; navigation surfaces were not in scope and should not have
shipped.

═══════════════ Removed ═══════════════

`src/screens/ResultsScreen.jsx`:
  • CATEGORY_NAV constant (Housekeeper / Pet Sitter / Handyman /
    Trainer / Driver — 5 entries).
  • The category-nav horizontal-scroll row above the filter pills.
  • The filter pill row (All services / Free / Discounted) — Free
    toggling stays on Home via the canonical "Free for Connectors"
    entry; "Discounted" was a visual-only placeholder anyway.

`src/screens/HomeScreen.jsx`:
  • The `seedQuery` / `seedFreeServices` location-state consumer
    useEffect was the only call site for these props (the rail's
    `navigate('/', { state: { seedQuery } })` and the Free pill's
    `state: { seedFreeServices }`). With the rail/pills gone, this
    wiring becomes dead code; removing it keeps Home's startup
    effects minimal.

═══════════════ NOT touched (out of scope) ═══════════════

  • Provider Card mint reco pill (3d) — stays. It replaced an
    italic inline aside, not new navigation.
  • Bottom "Bring your people · earn together" promo (3d) — stays.
    It replaced existing affordances rather than introducing new
    navigation.
  • Scale-token sweep on Results / ProviderCard (3d) — stays. That
    was the actual typography-audit work and is non-architectural.
