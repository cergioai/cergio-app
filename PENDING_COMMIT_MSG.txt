Service-area polygon — Zillow-style freehand draw on Google Maps

Tarik: "enable services to draw on a map to choose their service area
(perhaps borrow from how users choose property search areas on
Zillow…). apply this on the services IG homepage instead of the
radius… use Google and have quality Zillow-like experience."

Built: a bottom-sheet map picker that opens from the spotlight intent
on HomeScreen. Provider drags their finger across the map and a green
polygon fills under their gesture in real time. Release → polygon
commits. Tap Clear to redraw. The 5/10/25mi pills remain as a
"quick-pick" fallback for providers who don't want to draw, but they
dim out once an explicit polygon is set (the polygon takes precedence).

WHY CUSTOM FREEHAND
google.maps.drawing.DrawingManager only ships click-to-vertex polygons
— the user clicks individual corners and double-clicks to close.
That's not the Zillow experience. Zillow draws as you drag. So:
  • mousedown/touchstart → freeze map drag + zoom, start collecting
    LatLng points via the OverlayView's projection
  • mousemove/touchmove → push points (throttled by min-distance so
    we don't drown the polygon in noise), redraw an in-progress
    Polyline so the user sees their line forming
  • mouseup/touchend → close the polygon, drop a fill, restore map
    drag + zoom, fire onChange with the GeoJSON shape

OUTPUT SHAPE
GeoJSON Polygon:
  { type: 'Polygon', coordinates: [[[lng,lat], [lng,lat], ..., [lng,lat]]] }
First and last coordinates are equal (closed ring per spec). Plays
nicely with PostGIS for future server-side filtering.
