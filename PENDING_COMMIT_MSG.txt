═══════════════ (a) requests RLS — fixed via .command ═══════════════

Audit Notify Chain.command + Audit Fresh Request.command + Audit
Requests RLS.command together surfaced the cause: `requests`
has RLS enabled and only two SELECT policies — "request visible
to requester + target" and "self read" — both of which require
auth.uid() = requester_id OR target_provider_id. Broadcast
providers (matching service_type, no specific target) are excluded
by design, so listInboundRequests returned 0 rows even though
service_type='Personal Chef' exact-matched info@cergio.ai's
listed taxonomy_provider_type.

Apply Requests Read Policy.command adds:

  CREATE POLICY "matching providers can read pending requests"
  ON public.requests FOR SELECT TO authenticated
  USING (
    status = 'pending'::request_status
    AND created_at >= NOW() - INTERVAL '60 minutes'
  );

Net behavior: any authenticated user can read open requests
created in the last 60 minutes. listInboundRequests then filters
client-side by service_type → the provider's listed taxonomy.

Privacy: requester_id is a uuid; no email / address / payment
info on the row. Same posture as Indeed / Thumbtack — open
requests are broadcast to matching providers.

Verified: Section 4 of the policy .command returned the two
pending requests info@cergio.ai needs to see.

═══════════════ (b) BrowseConnectorsScreen confirmation gate ═══════════════

  • Polls `listMyOutboundSpotlightRequests` every 5s.
  • Builds confirmedConnectorIds = Set of connector_ids on rows
    with status IN ('offered', 'countered', 'accepted').
    A countered paid price still counts — "yes, here's my price"
    IS a confirmation per Tarik.
  • Renders Connectors filtered to confirmedConnectorIds under
    a new "Confirmed availability" header.
  • Two new gated states:
      – No outbound request ever sent → CTA prompting the user
        to list a service / start a broadcast.
      – Outbound request(s) exist but no Connector has confirmed
        yet → "Awaiting Connector responses…" state.
  • Free / paid banner copy now reflects the confirmation context:
    "No free Connectors confirmed yet" instead of "available."
