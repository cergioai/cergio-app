feat(profile): clickable avatars + public profile view (/u/:profileId)

Tarik feedback: "make all the avatars/profiles etc on the profile of
services (recommenders avatars) clickable to the users (showing their
recommendations)... replace Go-To with Reco'd or endorsed".

New screen:
  src/screens/PublicProfileScreen.jsx — modeled on the 7 attached
  Jennifer Leighton / Jacob Flores mockups. Sections (all from real
  DB rows; empty sections collapse silently — no fake data):
    • Header: large avatar + name + role badge + Connector badge
    • About: bio prose, "No bio yet!" empty state
    • Social: IG handle + follower count (or grey "Instagram" when
      not connected)
    • {Name}'s Services: services they own, opens PDP on tap, with
      "Reco'd by N friends and N Connectors" pill on each card
    • People who love {Name}: review rows pulled from bookings+reviews
    • {Name}'s Go-Tos: services this profile has recommended, joined
      to owners; tap message → service PDP, tap owner avatar → owner
      profile

Routing:
  src/App.jsx — new <Route path="/u/:profileId" /> + HIDE_NAV prefix
  so the profile renders without the bottom nav (modal-sheet style
  matching the mockups).

Clickable avatars throughout the app:
  - ServiceDetailScreen (PDP) — recommender AvatarStack, lead-recommender
    avatar, recommender-blurb avatars, provider-name header → all Links
    to /u/{id}. Per Tarik: "make all the avatars profiles etc on the
    profile of services".
  - ActivityScreen GoatShareCard — service-owner avatar + recommender
    pill avatar + recommender name in "Shared by X" → all Links to
    /u/{id}. Owner avatar Link uses stopPropagation so it doesn't fire
    the wrapping PDP-tap button.
  - ResultsScreen ProviderCard — FriendAvatars now renders Links when
    recommendersRaw is provided. Falls back to the old name-only avatars
    for any legacy/mock paths.

PDP copy:
  ServiceDetailScreen reco line: "Go-to service for N users, F friends
  and E experts, including Jennifer" → "Reco'd by F friends and E
  Connectors, including Jennifer". Comment block updated to match.

API:
  lib/api.js listGoatShares — adds service.owner_id to each row so the
  GoatShareCard owner avatar can link to /u/{ownerId}.
