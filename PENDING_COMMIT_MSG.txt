THEN Tarik tested live and the chat looped on the budget question
because Claude's parser doesn't extract bare numeric replies as
budgets. User typed "400", parser returned budget=null, gate asked
again. THE FIX:

  applyParseResult now has a bare-number capture step BEFORE merged
  is built. Detects "we were asking about budget" purely from
  prevState (what + when + where all set, budget missing) — no
  stale-closure on lastBotReply. When detected:
    • bare number like "400" or "$400"  →  fields.budget = "$400"
    • flex words (flexible|any|skip|none|no max|nope|na)  → "flexible"

  budgetSatisfied accepts a digit OR flex/any/none/no max/skip so
  both paths unblock the search. Phase 'ready' requires it too.
