UX: leaf intensity wired through CSS vars + AuthScreen trim

LEAF INTENSITY — REAL ACTIVITY DRIVES THE PLANT

LeafLogo now accepts an `intensity` prop (0..1). The keyframes in
src/index.css consume two CSS variables, --cg-leaf-amp + --cg-leaf-
speed, that scale sway amplitude and loop duration:

  intensity 0   → still
  intensity 0.4 → soft breathe (amp 0.72, slow speed)
  intensity 1.0 → full sway + fast sap flow

LeafLogo computes amp + speed from intensity and injects them as
inline CSS variables. cg-leaf-alive + cg-leaf-vein consume both via
calc() so a single intensity value reshapes the whole motion.

Wired in ResultsScreen: the loading-state leaf reads
`liveStatus.intensity` from useRequestActivity, so the plant
literally reacts to real notification + bid counts on the open
request. As replies land, the leaf sways harder and the sap flows
faster — the user can SEE that the network is engaging.

The intensity formula (canonical, in activityToStatus):
  min(1, (notified + replied * 3) / 10)
Bids count 3× because a reply is real engagement, not just a ping.

Backwards compatible: existing callers using `working={true|false}`
work unchanged — `working` becomes a 0/1 intensity alias when no
explicit intensity is passed.
