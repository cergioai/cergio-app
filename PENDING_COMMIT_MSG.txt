a top-right counter that locks the dollar amount in the eye:

  Step 1 — DIRECT CASH        — You invite, friend books, you earn 7% up to $250

  Step 3 — BARTER             — Connectors trade IG/TikTok spotlights for

  Step 4 — GROWTH PARTICIPATION — Community-owned upside as Cergio scales
                                 Tagline: Human-Powered AI · Shared Prosperity

4. SEARCH BUG — "need houskeeper under 200" returned empty state

Tarik's tab: "need houskeeper under 200" produced "What service do you
need? (e.g. plumber, sitter, cleaner, tutor…)" — even though local
taxonomy DOES resolve "houskeeper" → "House Cleaner" via the fuzzy
fallback (distance 1, well within threshold).

Root cause: useChat's bot-reply branch keyed on `merged.what` alone.
Claude's cloud parser returned what=null (typo + actor-noun confused it);
the LOCAL override at line ~520 correctly set provider_type, but the
reply logic ignored it and fell into the empty-state path.

  src/hooks/useChat.js
    • Backfill merged.what from mergedProviderType when cloud parser
      returns null but local taxonomy resolved
    • Bot-reply gate now checks `merged.what || merged.provider_type`
      so a resolved provider type is enough to skip the empty-state prompt

  scripts/qa.mjs #29 (search-tolerance)
    • +5 cases locking actor-noun typos:
        'houskeeper'                  → House Cleaner
        'need houskeeper under 200'   → House Cleaner   (user's exact phrase)
        'housekeping under 200'       → House Cleaner
        'plummer'                     → Plumber
        'plumbr needed today'         → Plumber
        'electrcian needed urgently'  → Electrician
    • Brings the search tolerance battery to 70+ cases
