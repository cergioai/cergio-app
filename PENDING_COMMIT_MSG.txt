Two parts: (a) the reco line bucketing, (b) the budget input that lets
the Saves $X / Over budget $X labels actually render.

1. RECO LINE — bucketed format

Old format: "Reco'd by Alex, Sam, Connie" (flat list of all names).
New format: "Reco'd by {leadFriend}, {N} other friends and {M} Connectors"
where the named anchor is the first NON-Connector friend (if any) and
Connectors are surfaced separately so the social-proof reads cleanly.

Data layer:
  src/lib/api.js — fetchRecommendersByServiceId
    • SELECT now includes profiles.cc_verified_at
    • Each recommender carries is_connector: !!p?.cc_verified_at

Shape layer:
  src/screens/ResultsScreen.jsx — serviceToProvider
    • Buckets svc.recommenders into friendsRaw (non-Connector) and
      connectorsRaw (cc_verified)
    • Exposes friendCount, connectorCount, leadFriendName on the
      provider object
    • Accepts friendOwnerIds for future "in your network" filtering

Render layer:
  src/components/ui/ProviderCard.jsx — recoText()
    • Builds the natural-language string:
      "Reco'd by Jennifer Hu, 3 other friends and 21 Connectors"
      "Reco'd by Jennifer Hu and 21 Connectors"      (1 friend + Cs)
      "Reco'd by 21 Connectors"                       (no friends)
      "Reco'd by Jennifer Hu"                         (1 friend, no Cs)
    • Legacy fallback preserved for old codepaths that still set the
      flat `friends` array without buckets

  scripts/qa.mjs #40 (new)
    • Locks the is_connector derivation in api.js
    • Locks the friendCount + connectorCount + leadFriendName fields
    • Locks the "other friend" + "Connector" wording in ProviderCard

2. HOME BUDGET PILL — "flexible" by default

Mirrors the date "I'm flexible" pattern. New optional Budget input
sits inline in the search controls row, between the attach button
and the Free/Pay mode toggle:

  $ [____]   (placeholder reads "flexible" when empty)
  $ [200]  ×  (clear button appears once a value is typed)

Empty input = no budget filter (flexible). Typing a number flows
into the existing submit pipeline as budget_cents, which the
ResultsScreen savings calculator already uses to render
"Saves $X" or "Over budget $X" labels per card.

The manual budget wins over whatever the chat parser inferred
from free text. (Old behaviour stays — "under $200" in chat text
still works as a fallback when the explicit input is empty.)

  src/screens/HomeScreen.jsx
    • New `budget` state (string)
    • Inline budget input pill in the search controls row
    • submitQuery prefers manualBudget over s.budget for budget_cents
