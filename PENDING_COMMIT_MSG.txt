═══════════════ Phase 4 — interaction polish (§ 5.7) ═══════════════

Built as a single `@layer components` block in `src/index.css` so
every CTA/tap-target picks up consistent hover/active/focus without
re-deriving the same Tailwind stack each time.

Components
  .cg-cta        — primary fill (Sign in, Book, Next, Forward,
                   Invite friends, Reviews CTA…)
                   hover:opacity-90 · active:scale-[.97]
                   focus-visible:ring-[3px] #F3FFEA
  .cg-cta-ghost  — outline/secondary (Become a Connector, Copy,
                   Back, "Leave a go-to review"…)
                   hover:bg-bg5 · active:scale-[.98] · same ring
  .cg-tap        — card-like tap-targets (ServiceTile, Go-Tos
                   quote bubble, ProviderCard photo)
                   hover:bg-bg5/40 · active:scale-[.99] · same ring
  .cg-focusable  — focus-only mixin for inline text-button affordances
                   (Continue as guest, EarnExplainer ✕)

Applied to
  AuthScreen           — Sign in (cg-cta), Continue as guest (cg-focusable)
  ProviderCard         — Book button (cg-cta / cg-cta-ghost), photo (cg-tap)
  PublicProfileScreen  — ServiceTile (cg-tap), Go-Tos quote (cg-tap)
  EarnExplainerScreen  — ✕ close (cg-focusable)
  RewardFlowAnimation  — Back (cg-cta-ghost), Next/Replay (cg-cta)
  ResultsScreen        — primary buttons swept via Python pass to
                         drop inline hover:opacity-90 + active:scale
                         in favor of .cg-cta / .cg-cta-ghost classes
                         (1 cg-cta + 1 cg-cta-ghost upgrade).

Also added .no-scrollbar (sibling of .scrollbar-hide) since the
category-nav rail in Results uses it.

═══════════════ Phase 5 — screenshot QA harness (§ 5.8) ═══════════════

Cowork sandbox can't run dev server + headless browser against the
user's localhost, so capture is driven on-device via a one-click
script.

  Screenshot QA.command (sibling of Unlock and Push.command)
    • Lazy-installs Playwright + Chromium under ~/.cergio-playwright
      (one-time, ~80MB)
    • Starts `npm run dev` if not running
    • Runs the capture script
    • Opens the gallery in the default browser

  cergio-app/scripts/screenshot-qa.mjs
    • Visits 10 canonical routes at 390 × natural viewport, 2× DPR
    • Writes design-qa/{key}.png per screen
    • Generates design-qa/index.html — side-by-side gallery with
      live capture + reference mockup (when a canonical PNG exists)
    • Rows keyed by audit § so the layout mirrors the punch list
    • § 7's 2-px vertical-baseline tolerance called out in header
