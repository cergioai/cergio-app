Activity feed — GOAT shares (real data, hides when empty)

Tarik: "tweak activity feed to show something like this (friends and
connectors booking, reco'ing etc.)" with the GOAT social-feed mockup.

Approach:
A clean section ABOVE "Your open requests" titled
  "GOATs have shared their go-to services on Cergio"
  #cergiogoats
that renders cards from REAL `recommendations` rows authored by users
with cc_verified_at NOT NULL (Connectors / GOATs). Empty section just
hides — zero fake data, zero shame state. Consistent with the
established "no mock data on signed-in screens" rule
(feedback_no_fake_feeds in memory; the prior fake "Friends recently
booked" FEED was removed twice already and would have silently
regressed without this discipline).

Each card matches the mockup:
  • Big provider name + "was shared on Cergio"
  • Category pill (green check) · city (location_text)
  • 140px cover image (real cover_url) with photo_class gradient
    fallback when none
  • Bottom pill: "Shared by {Connector Name}, GOAT" with small avatar
Tap → routes to the provider's PDP (/service/:id).

What we deliberately do NOT show:
  • "Shared to N followers" — we don't track follower counts on
    profiles; would have had to fake the number. Omitted entirely.
  • Multiple media thumbnails per card — we don't yet have a
    multi-image gallery on services. Single cover is the honest
    representation of what we actually have.

Plumbing:
  src/lib/api.js
    + listGoatShares({ limit }) — multi-step fetch:
      1. recent recommendations (limit×3 to overfetch)
      2. recommender profiles → keep cc_verified_at NOT NULL
      3. services for kept recs
      4. service-owner display names ("Sabir was shared…")
      5. shape + slice to limit, return [] on empty

  src/screens/ActivityScreen.jsx
    + GoatShareCard component
    + new top section, gated on goatShares.length > 0
    + loads listGoatShares on mount, signed-in OR signed-out (the
      recommendations RLS policy is `select using (true)` so the
      feed is public-ish; matches the mockup's discovery intent)
