fix(profile): blank /u/{id} page + bulletproof loading + tagline switch

Two fixes in one commit:

1. Public profile rendered blank
   Tarik: "click link goes to blank page (populate even seeded
   profiles..)".

   Root cause: the profile SELECT included `avatar_url`, which doesn't
   exist on the profiles table. PostgREST 400'd the query → prof=null
   → notFound branch fell through silently in some HMR states.

   Fix:
   - Drop `avatar_url` from the SELECT (no other code reads it; we
     render initials avatars instead).
   - Console.warn on profile / reviews fetch errors so future drift is
     visible.
   - Loading + notFound branches now render the X close button + cream
     background, so the user is never on a truly blank scroll area.
   - Reviews query rewritten to use reviews.rater_id directly (was
     hopping reviews → bookings.consumer_id, which is wrong — the
     reviewer is rater_id).

2. Tagline switched on Splash + Auth (sign-up) screens
   Tarik: "Cergio AI negotiate and book services your friends actually
   trust. Book, Barter, Earn... or a succinct variation."

   Picked the succinct rewrite — the wordmark above the tagline
   already says CERGIO, so we don't repeat the brand:

     AI that negotiates and books services your friends actually trust.
     Book · Barter · Earn

   Applied to SplashScreen (replaces "Find trusted services,
   recommended by friends") and AuthScreen (replaces "Services your
   friends actually trust. Book, host, or earn — all in one place.").
   Sub-line uses tracking-[0.18em] uppercase to read as a verb stamp.
