═══════════════ (1) Earnings potential math fixed ═══════════════

  • Per friend potential = perFriend + friendOfFriendBonus = $262.50
    (one direct cap + one chain bonus they unlock).
  • potential = max(perFriend × invitesCount, REWARDS.exampleTotal)
    so new users see Jamie's $12,500 example as the floor, scaling
    higher when actual invites push past it.
  • Hero text: "$838 of $12,500 potential" with progress bar.

═══════════════ (2) Invite funnel — Invited → Joined → Booked ═══════════════

  • New getMyInviteCounts() API helper reads invites lifecycle
    timestamps (invited_at always set; joined_at on signup;
    first_booking_at on first paid booking) and returns
    { invited, joined, booked }.
  • ReferralsSummary renders the three counts as side-by-side
    cards with arrows between, matching the typical funnel UI.
  • Stalled = invited − joined. When > 0, an inline green
    "Re-invite N friends who haven't joined →" link routes
    to /invite/friends so the user can nudge them back.
